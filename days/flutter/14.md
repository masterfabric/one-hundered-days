## ğŸ“– Task Explanations

A `Future` represents a single value that will be available in the future. A `Stream`, on the other hand, represents a sequence of asynchronous events. You can think of a `Stream` as an asynchronous `Iterable`. Instead of getting a single value, you can listen to a `Stream` and receive a series of values over time.

Streams are used for many things in Dart and Flutter, such as handling user input events, reading data from a file, or receiving data from a WebSocket. Today, you'll learn the basics of how to create and listen to streams.

## ğŸ“š Dictionary for Developers

*   **`Stream`:** A sequence of asynchronous events. It's like an asynchronous `Iterable`.
*   **`StreamSubscription`:** An object that represents a subscription to a `Stream`. It can be used to manage the subscription (e.g., pause, resume, cancel).
*   **`listen()`:** A method on a `Stream` that you call to subscribe to the stream's events.
