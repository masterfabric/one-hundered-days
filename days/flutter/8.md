## üìñ Task Explanations

Today is about defining contracts for your classes. An `abstract class` is a class that cannot be instantiated. It's used as a base class for other classes. It can have abstract methods, which are methods that are declared but not implemented. The subclasses are then forced to provide an implementation for these abstract methods.

In Dart, every class implicitly defines an interface. An interface is a contract that a class can choose to fulfill. When a class `implements` an interface, it must provide an implementation for all the methods and properties defined in that interface. This is a powerful way to write decoupled code, where different parts of your application can communicate with each other through a common interface, without needing to know about the specific implementation details.

## üìö Dictionary for Developers

*   **Abstract Class:** A class that cannot be instantiated and is meant to be subclassed. It can contain abstract methods (methods without a body).
*   **Interface:** A contract that a class can implement. In Dart, any class can be used as an interface. When a class implements an interface, it must provide an implementation for all of the interface's methods and properties.
*   **`implements`:** A keyword in Dart used to implement an interface. A class can implement multiple interfaces.
