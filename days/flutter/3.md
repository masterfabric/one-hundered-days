## üìñ Task Explanations

Null safety is one of Dart's most powerful features for preventing a common type of error: the null reference exception. Today, you'll dive deep into this concept. You'll learn to explicitly tell the Dart compiler which variables are allowed to be `null` and which are not. This allows the compiler to catch potential errors at compile-time, before your app even runs. You'll learn the syntax for handling nullable types safely, which will make your code more robust and reliable.

## üìö Dictionary for Developers

*   **Null Safety:** A feature in Dart that helps you avoid errors that result from unintentionally trying to use a variable that has a `null` value.
*   **Nullable Type:** A type that can hold a `null` value in addition to its regular values. It's denoted by a `?` after the type name (e.g., `int?`).
*   **Non-nullable Type:** A type that cannot hold a `null` value. By default, all types in Dart are non-nullable.
*   **Null-aware Operators:** Operators that allow you to work with nullable variables in a safe way. For example, `?.` lets you access a property or method on an object only if the object is not `null`.
*   **`late`:** A keyword that tells Dart you will initialize a non-nullable variable before it's used, even though you're not initializing it at the time of declaration.
