## üìñ Task Explanations

While `.then()` and `.catchError()` are powerful, they can sometimes lead to code that is hard to read, especially when you have multiple nested asynchronous operations (a situation sometimes called "callback hell").

`async` and `await` are syntactic sugar that make your asynchronous code look and behave a lot more like synchronous code. When you mark a function with the `async` keyword, you can use the `await` keyword inside it. `await` pauses the execution of the function until the `Future` you're awaiting completes. It then unwraps the value of the `Future` and resumes the function's execution. This allows you to write asynchronous code in a more linear and readable fashion.

## üìö Dictionary for Developers

*   **`async`:** A keyword in Dart used to mark a function as asynchronous. An `async` function automatically returns a `Future`.
*   **`await`:** A keyword in Dart that can only be used inside an `async` function. It pauses the execution of the function until the `Future` it's awaiting completes.
