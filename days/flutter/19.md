## ğŸ“– Task Explanations

A `StatefulWidget` is a widget that has mutable state. This means its appearance can change during the lifetime of the widget, for example, in response to user interaction. A `StatefulWidget` is actually composed of two classes: a `StatefulWidget` and a `State` object. The `State` object is where the state of the widget is stored.

When the state of a widget changes, you need to call the `setState()` method. This tells Flutter that the widget is "dirty" and needs to be redrawn. Flutter will then call the `build()` method of the widget to update its appearance.

Today, you'll also get a glimpse into the lifecycle of a stateful widget. You'll learn about the `initState()` method, which is called once when the widget is first created, and the `dispose()` method, which is called when the widget is permanently removed from the widget tree.

## ğŸ“š Dictionary for Developers

*   **`StatefulWidget`:** A widget that has mutable state.
*   **`State`:** An object that holds the mutable state for a `StatefulWidget`.
*   **`setState()`:** A method that notifies Flutter that the internal state of a widget has changed.
*   **`initState()`:** A method that is called once when a `State` object is first created.
*   **`dispose()`:** A method that is called when a `State` object is permanently removed from the widget tree.
