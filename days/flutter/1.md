## üìñ Task Explanations

Today is about laying the groundwork for your journey into Dart and Flutter. A proper development environment is essential for a smooth learning experience. You'll learn not just how to store data in variables, but also the important distinction between variables that can be reassigned (`var`) and those that are meant to be constant (`final`, `const`). Using `final` and `const` appropriately is a best practice in Dart that leads to more robust and predictable code.

Finally, you'll get to know the basic building blocks of data in Dart: numbers (`int`, `double`), text (`String`), and boolean values (`bool`). Every piece of data in your application will be composed of these fundamental types.

## üìö Dictionary for Developers

*   **SDK (Software Development Kit):** A collection of software development tools in one installable package. The Dart SDK includes the Dart compiler, libraries, and a runtime environment.
*   **Variable:** A named storage location in a computer's memory that holds a value. In Dart, variables are declared with keywords like `var`, `final`, or `const`.
*   **Data Type:** A classification that specifies which type of value a variable can hold. Examples include `int` for integers and `String` for text.
*   **`var`:** A keyword in Dart used to declare a variable whose type is inferred by the compiler and whose value can be reassigned.
*   **`final`:** A keyword in Dart used to declare a variable that can only be set once. It's a runtime constant.
*   **`const`:** A keyword in Dart used to declare a variable that is a compile-time constant. This means its value must be known at the time the code is compiled.
