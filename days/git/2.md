## ğŸ“– Task Explanations

Saving changes in Git is a two-step process. First, you add the changes you want to save to the "staging area" using `git add`. This allows you to group related changes together. Second, you commit the staged changes to the repository's history with `git commit`. This creates a permanent snapshot of your project at that point in time.

*   **`git add`:** A command that adds changes from the working directory to the staging area.
*   **`git commit`:** A command that saves the changes from the staging area to the repository's history.
*   **Staging Area:** An intermediate area where you can prepare changes before committing them.
*   **Commit:** A snapshot of your project at a specific point in time.
