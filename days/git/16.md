Tags are used to mark specific points in your repository's history as being important. Typically, people use this functionality to mark release points (e.g., `v1.0`). Tags are like a branch that doesn't move. Once a tag is created, it stays on that commit forever, making it easy to find and check out specific versions of your project in the future.
