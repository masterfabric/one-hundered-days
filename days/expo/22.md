## üìñ Task Explanations

In React Native, you don't use HTML `div`s and `p` tags. Instead, you use native components that map directly to platform-specific UI elements. `View` and `Text` are the two most fundamental of these. Think of `View` as a `div` and `Text` as a `p` tag, but they are much more powerful and optimized for mobile performance.

`View` is a container that supports layout with Flexbox, styling, touch handling, and accessibility controls. `Text` is specifically designed for displaying text and supports nesting for different styles within a single block of text. Today, you'll get comfortable using these two core components and applying basic styles to them, which is the starting point for any React Native UI.

## üìö Dictionary for Developers

*   **`View` (React Native):** The most fundamental component for building UI. It's a container that supports layout with Flexbox, styling, touch handling, and accessibility controls.
*   **`Text` (React Native):** The component used to display text in a React Native application.
*   **`StyleSheet` (React Native):** An abstraction similar to CSS stylesheets. It provides a way to define styles in a centralized, optimized manner.
