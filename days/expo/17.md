## üìñ Task Explanations

Props are the primary mechanism for communication between components in React. They allow you to make your components dynamic and reusable. Think of props as arguments you pass to a function; in this case, the function is your component, and the arguments are the data it needs to render.

Today, you'll get hands-on with passing data down the component tree. You'll also integrate your TypeScript knowledge by defining `Prop Types` using interfaces. This is a crucial step for building robust React applications, as it helps catch errors related to incorrect data types being passed to components. Setting default props provides a fallback for when a parent component doesn't supply a particular prop, making your components more resilient.

## üìö Dictionary for Developers

*   **Props (Properties):** An object that contains data passed from a parent component to a child component. Props are read-only.
*   **Prop Types (TypeScript):** Using TypeScript interfaces to define the expected types of props that a component should receive, enabling static type-checking.
*   **Default Props:** Default values assigned to props that are not explicitly provided by the parent component.
