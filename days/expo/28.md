## üìñ Task Explanations

While React Native uses a syntax similar to CSS for styling, it's important to understand that it's not exactly CSS. React Native styles are JavaScript objects, and they are applied directly to components. The `StyleSheet.create()` method is the recommended way to define your styles because it optimizes them by sending them to the native side only once, improving performance.

Today, you'll learn the differences between inline styles (defining styles directly within the component's JSX) and using `StyleSheet.create()`. You'll understand why `StyleSheet.create()` is generally preferred for complex applications. You'll also learn how to combine multiple style objects, which is very useful for creating reusable style patterns and overriding styles when needed.

## üìö Dictionary for Developers

*   **`StyleSheet` (React Native):** An abstraction similar to CSS stylesheets. It provides a way to define styles in a centralized, optimized manner.
*   **`StyleSheet.create()`:** A method from `StyleSheet` that creates a style object from a given JavaScript object. It performs optimizations by assigning unique IDs to style objects.
*   **Inline Styles:** Styles defined directly within the `style` prop of a React Native component, as a JavaScript object.
*   **CSS-in-JS:** A pattern where CSS is authored using JavaScript. React Native's styling approach is a form of CSS-in-JS.
