## üìñ Task Explanations

Interfaces are a cornerstone of TypeScript, allowing you to define custom types that describe the shape of objects. They act as contracts, ensuring that any object or class that claims to be of a certain interface type will have all the specified properties and methods. This is incredibly powerful for maintaining consistency and catching errors early in your development process.

Today, you'll learn the syntax for defining interfaces, including how to make properties optional or read-only. You'll also see how interfaces can extend each other, promoting code reuse and creating more complex type definitions. This structured approach to data will make your code much more predictable and easier to work with, especially as your React Native applications grow in complexity.

## üìö Dictionary for Developers

*   **Interface (TypeScript):** A powerful feature that defines the syntax that any entity must adhere to. It's a contract for the shape of an object.
*   **Optional Property:** A property in an interface that does not necessarily have to be present in an object that implements the interface. Denoted by `?`.
*   **Read-only Property:** A property in an interface that can only be assigned a value when an object is first created. Denoted by `readonly`.
*   **Extending Interfaces:** The ability for one interface to inherit properties and methods from another interface.
