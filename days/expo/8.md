## üìñ Task Explanations

While Promises significantly improved asynchronous JavaScript, `async` and `await` take it a step further by allowing you to write asynchronous code that looks and behaves almost exactly like synchronous code. This dramatically improves readability and makes complex asynchronous flows much easier to reason about.

When you declare a function as `async`, it signals that the function will perform asynchronous operations and will always return a Promise. Inside an `async` function, you can use the `await` keyword before any Promise-returning expression. `await` will pause the execution of the `async` function until that Promise resolves, and then it will return the resolved value. If the Promise rejects, `await` will throw an error, which you can catch using a standard `try...catch` block.

## üìö Dictionary for Developers

*   **`async`:** A keyword used to define an asynchronous function. An `async` function always returns a Promise.
*   **`await`:** A keyword that can only be used inside an `async` function. It pauses the execution of the `async` function until the Promise it's waiting for settles (resolves or rejects).
*   **`try...catch`:** A JavaScript statement that allows you to test a block of code for errors (`try`) and handle them gracefully (`catch`). Essential for error handling in `async/await`.
