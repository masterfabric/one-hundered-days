## üìñ Task Explanations

While `ScrollView` is great for a small number of items, it renders all its children at once, which can lead to performance issues with very long lists. `FlatList` is designed specifically for efficiently rendering long lists of data by only rendering items that are currently visible on the screen, a technique called "virtualization."

Today, you'll learn the core props of `FlatList`: `data` (your array of items), `renderItem` (a function that tells `FlatList` how to render each item), and `keyExtractor` (a function that provides a unique key for each item, crucial for performance). You'll also explore how to add visual elements like separators between items or headers/footers for the entire list.

## üìö Dictionary for Developers

*   **`FlatList` (React Native):** A component for rendering long lists of data efficiently. It only renders elements that are currently showing on screen, not all the elements at once.
*   **`data` (FlatList prop):** The array of data that `FlatList` will render.
*   **`renderItem` (FlatList prop):** A function that takes an individual item from the `data` array and returns a React component to render for that item.
*   **`keyExtractor` (FlatList prop):** A function that takes an item and its index and returns a unique string identifier for that item. This helps React Native optimize list rendering.
